比赛1：Amazon项目¶

【摘要】¶

本文是对Kaggle上Amazon项目作的一次数据分析报告，目的是通过已知的Amazon公司员工的职位信息去预测该员工是否能被批准利用公司的某些资源（二分类问题）。本文应用的特征提取方法是One-Hot Encoding（独热编码）， 建模方法包括Logistic回归和随机森林。分析发现，逻辑回归如果不进行调参，准确率会很低。于是又分别进行了Lasso和Ridge两种正则化方法，并对不同的惩罚项系数C进行了选择。而随机森林又分为随机森林分类和随机森林回归，结果发现，若采取随机森林回归去求解该分类问题，准确率反而更高。

#定义一个交叉验证函数
SEED = 22
N=10
def cv_loop(X, y, model, N):#N折交叉验证
    mean_auc = 0.
    for i in range(N):
        #80%用于训练，20%用于测试
        X_train, X_cv, y_train, y_cv = cross_validation.train_test_split(
                                       X, y, test_size=.20, random_state = i*SEED)
        model.fit(X_train, y_train)
        preds = model.predict_proba(X_cv)[:,1]
        #计算AUC值
        fpr, tpr, thresholds = metrics.roc_curve(y_cv, preds)
        auc=metrics.auc(fpr, tpr)
        mean_auc += auc
    return mean_auc/N

比赛2：数字识别¶

【摘要】¶

本文是对Kaggle上Digit Recognizer项目作的一次数据分析报告，目的是通过数字在画布上呈现出的28×28的像素值去判断该数字属于0到9中的哪一个（多分类问题）。在特征工程上，为简便起见，本文首先对每个数字的像素大小作了二值化处理（0和1），并将784个像素值转化为28×28的矩阵，然后从数值矩阵中提取了每四行的和、每四列的和、16个7×7的小矩阵的和、最大行和、最大列和、中间6×6的矩阵和、第一个1所在列以及最后一个1所在列一共35个特征。紧接着，本文应用Lasso、Ridge和K邻近三种机器学习方法对这35个变量进行了建模预测，发现不仅能够大幅提高运算效率，准确度也不低（91.7%）。
